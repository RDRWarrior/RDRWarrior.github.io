Lodestar

An offline-first convention planning app built to help attendees browse events, manage schedules, and navigate venues in a live event environment.

Framework: Flutter
Language: Dart
Category: Convention Planning App
Focus: Offline-First UX

Overview

Lodestar is a convention planning app designed to help attendees browse schedules, build personal itineraries, and navigate event spaces without depending on constant internet access. The project is built around the realities of live conventions, where fast access, clear organization, and reliable offline functionality matter more than always-online features.

My Role

I designed and developed the structure of the app, including event browsing, schedule planning, and venue navigation features. The project was shaped around practical user needs at large multi-room events, with an emphasis on clean UI flow and scalability.

Core Features

  • Offline-First Event Browsing: Users can access event and venue information without relying on live connectivity.
  • Personal Schedule Building: Attendees can favorite events and create their own in-app schedule.
  • Conflict Awareness: The app warns users when selected events overlap in time.
  • Venue Navigation: Map-based navigation helps users move between rooms and locations.
  • Search and Organization: Events can be organized through tags, categories, and searchable structure.

Technical Focus

The project focuses on offline-first application design, clean mobile UI architecture, and scalable event data handling. It is structured to remain useful during high-friction event conditions, where connectivity may be limited and users need quick, reliable access to information.

Design Challenges

A central challenge was balancing feature depth with usability. Convention apps can become cluttered quickly, so the design needed to present schedules, locations, and personal planning tools in a way that remained fast to read and easy to navigate during a live event.

Development Status

Lodestar is an active app project focused on event browsing, schedule management, and venue navigation. The current work demonstrates strong foundations in data structure, user flow, and feature planning, with room for future expansion into richer map support, vendor information, and additional convention tools.
