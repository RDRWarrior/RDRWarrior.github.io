Lodestar

An offline-first convention planning app built in Flutter to help attendees browse events, manage schedules, and navigate venues in a live event environment.

Framework Flutter
Language Dart
Storage Hive
Focus Offline-First UX

Overview

Lodestar is a convention planning app designed around the realities of live events: limited connectivity, multi-room schedules, overlapping sessions, and the need to move quickly between locations. Rather than relying on always-online features, the app is built to remain useful through local data, personal schedule tools, and venue-aware navigation.

My Role

I designed and implemented the app structure, event browsing flow, personal schedule features, and venue navigation experience. The project focused on creating a practical, user-facing tool that could support real convention use rather than just presenting schedule data.

Core Features

  • Offline-First Event Access: Event and venue information can be loaded and used without depending on constant connectivity.
  • Personal Schedule Building: Users can favorite events, create a personal schedule, and manage their own convention plans.
  • Conflict Detection: The app detects overlapping events and warns users when schedule choices clash.
  • Personal Event Creation: Users can add their own custom events into the schedule alongside convention programming.
  • Venue Navigation: Map and routing tools help users move between start and destination rooms.
  • Search & Filtering: Users can filter by day, location, and type, with fuzzy search support for easier event discovery.
  • Notifications: Reminder scheduling supports upcoming event awareness.

Technical Focus

Lodestar is structured around service-driven app architecture. Event loading, venue loading, favorites storage, user data, routing, and notification handling are separated into dedicated services, making the project easier to extend and maintain. Local persistence is handled with Hive, allowing the app to retain useful state between sessions.

On the UI side, the app uses a tab-based structure with dedicated event, schedule, and map views. I also implemented custom styling logic tied to room and world themes, allowing the schedule experience to feel more visually distinct than a standard list-based planner.

Design Challenges

One of the biggest challenges was balancing feature depth against usability. Convention schedules can become cluttered quickly, so the interface had to support filtering, schedule management, routing, and conflict awareness without becoming slow to read or cumbersome to navigate during a live event.

Another challenge was building around real-world edge cases, such as partial data, fuzzy searches, missing times, and user-created events that need to coexist cleanly with scheduled convention content.

Development Status

Lodestar is an active app project with strong foundations in offline-first event browsing, schedule management, and venue navigation. The current build demonstrates practical mobile architecture, structured data handling, and user-focused planning tools, with room for future expansion into richer map support, vendor modules, and additional convention utility features.
