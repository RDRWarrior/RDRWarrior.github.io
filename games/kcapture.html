Kandra Capture

Unity asymmetric social deduction prototype focused on disguise, evidence, and player tracking systems.

Overview

KCapture is a Unity-based asymmetric social deduction prototype where players blend into NPC crowds while hunters attempt to identify and capture them using evidence, abilities, and environmental cues.

My Role

Designed and implemented gameplay systems, interaction mechanics, UI flow, and core game loop logic.

Core Systems

  • Task and interaction systems for player objectives
  • Disguise and identity blending mechanics
  • Evidence and detection systems for hunters
  • Match flow and win condition logic
  • UI prompts and interaction feedback

Technical Focus

Focused on modular gameplay systems, readable interactions, and rapid iteration within Unity. Systems were designed to be extendable for future multiplayer integration.

Challenges

One of the main challenges was balancing readability and deception—ensuring players could blend in convincingly while still providing enough feedback for hunters to track behavior patterns.

Current State

The project is currently in an advanced prototype stage with core gameplay systems implemented and undergoing visual polish and system refinement.
